## 对话与行动记录

- 用户反馈：在 MiniMax 等 tab 修改过 API 地址后，切到 OpenAI 仍显示 MiniMax 的值。
- Agent 检查当前实现，确认已有 `AI-A-016` 的“占位值判断”修复，但状态模型仍然让所有 provider / preset 共用同一组 `endpoint`、`model`、`apiKey` 和 `prompt`。
- 根因：切换 tab 时只改变 `provider` / `preset` 字段，表单值仍来自同一套顶层字段；一旦用户手动修改，切到别的 tab 时就会继续显示上一组值。
- 修复方案：为 `AiSettings` 增加 `profiles`，按 `provider:preset-or-region` 保存独立表单值。
- 实现 `switchAiSettingsProfile()`：
  - 切换前保存当前 profile。
  - 切换后加载目标 profile。
  - 目标 profile 不存在时才使用对应 preset / DeepL region 默认值。
  - 初始空配置不会被保存为空 profile，避免空值压过 OpenAI 默认值。
- 将 OpenAI preset、Anthropic preset、DeepL provider 和 DeepL free/pro 切换都改为走 profile 切换。
- 增加回归测试，覆盖 MiniMax 自定义 endpoint 与 DeepL endpoint 不串到 OpenAI 的场景。
