## 对话与行动记录

- 与用户对齐意图：AI 设置（含 apiKey）持久化到 `localStorage`，本浏览器配一次下次自动恢复。
- 主动说明安全 trade-off：`localStorage` 是明文，本机任何能跑 JS 的脚本可读；Lingrid 是本地工具且无第三方脚本，实际可接受。placeholder 措辞会改为如实告诉用户存储位置。
- 在 `App.tsx` line 54 旁加 `AI_SETTINGS_KEY = "lingrid-ai-settings"`，与 `UI_LANGUAGE_KEY` 同段。
- 把 `const AI_DEFAULT` 改为 `export const AI_DEFAULT`（之前是 const 局部，单测无法 import）。
- 抽两个纯函数 `loadAiSettings(storage?)` 与 `saveAiSettings(ai, storage?)`，默认参数走 `window.localStorage` 但接受 `StorageLike` 注入，便于单测和未来切到 sessionStorage。
  - `loadAiSettings`：处理三种边界——空存储（返回 `AI_DEFAULT`）、缺字段（`{...AI_DEFAULT, ...parsed}` 兜底）、JSON 损坏（catch 返回 default）。
  - `saveAiSettings`：try/catch 吞 storage 异常（QuotaExceeded / 隐私模式禁用），失败不抛、不影响 UI。
- 改 `useState<AiSettings>(AI_DEFAULT)` 为 `useState<AiSettings>(() => loadAiSettings())`，与 `uiLanguage` 同样的延迟初始化模式。
- 加 `useEffect(() => { saveAiSettings(ai); }, [ai])`，与 `uiLanguage` 同样的同步写回模式。
- 改 OpenAI 兼容 API 密钥 placeholder 措辞：`"Stored only in this local session"` → `"Stored in this browser via localStorage"`。DeepL placeholder 保留 `DeepL-Auth-Key xxxx-xxxx-xxxx-xxxx` 不动（已经隐含了存储位置）。
- 加 6 个 vitest 单测覆盖 load/save：JSON 写入、round-trip、空存储、缺字段 fallback、corrupt JSON、storage 异常不抛。
