## 对话与行动记录

### Part 1 — 用户报告"切 tab 不切内容"

- 用户原话："切到 openai 显示的还是 minimax 的内容"——意思是切到 OpenAI 时 form 仍显示之前 MiniMax preset 的内容
- 但**当前 OpenAI 预设 group 是渲染的**（OpenAI / Kimi / MiniMax / 通义千问 / 豆包 / Gemini / Claude / 自定义 8 个）——所以"切 tab 不切"**不**指预设 group 不变
- 实际是 **endpoint / model input 内容没切**：从 playwright 现场看，切到 DeepL 自动填 `https://api-free.deepl.com/v2/translate`，切回 OpenAI 时 endpoint input 仍显示这个 DeepL endpoint
- 顶部 `已记住: OpenAI-compatible · https://api-free.deepl.com/v2/translate · MiniMax-M2.7 · key ✓` —— provider 是 OpenAI 但 endpoint 是 DeepL

### Part 2 — 根因

- 上轮我修了"切预设不覆盖用户值"，但**修过头**了：`selectOpenAiPreset` 改的 `endpointStillPlaceholder` 判断是 `ai.endpoint === previous.endpoint`（与**上一** preset 比）
- 切到 DeepL 自动填了 `https://api-free.deepl.com/v2/translate`，切回 OpenAI 时 `ai.endpoint === previous.endpoint` 为 false（DeepL endpoint ≠ OpenAI preset endpoint）→ 保留 DeepL endpoint
- 同样 DeepL radio onChange `setAi({...ai, provider: "deepl", endpoint: DEEPL_ENDPOINTS[...]})` —— **无脑覆盖** endpoint

### Part 3 — 修法

- 新建 `ts/renderer/providers/placeholder-detect.ts`，导出 `isKnownPlaceholderEndpoint` / `isKnownPlaceholderModel` 纯函数
- 判定逻辑：endpoint 是空 / DeepL free / pro 端点 / **任何** OPENAI_COMPATIBLE_PRESETS.endpoint / **任何** ANTHROPIC_COMPATIBLE_PRESETS.endpoint → 占位
- `selectOpenAiPreset` / `selectAnthropicPreset` / DeepL radio onChange 三处都改用 `isKnownPlaceholderEndpoint` / `isKnownPlaceholderModel`
- 行为：切 provider / 切 preset 时，**只有**当前 endpoint / model 是"已知占位"才覆盖成新默认；用户手填的真值（不在占位集合里）原样保留
- playwright 实测：填 `https://api.minimaxi.com/v1/chat/completions` → 切 DeepL → 切回 OpenAI → endpoint **保留** `https://api.minimaxi.com/...` ✅
