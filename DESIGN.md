# Lingrid 灵译视觉规范

`DESIGN.md` 只记录视觉设计系统，不记录系统架构、仓库说明或具体需求。系统说明写入 `README.md`，任务和验收写入 `REQUIREMENTS.md`。

## 视觉主题

- 关键词：现代、轻量、安静、高密度、清晰、可信、本地化工具、矩阵编辑。
- 整体气质：参考 Linear 的克制产品气质，结合 shadcn/ui 的组件语言。界面应像专业开发工具，而不是 SaaS 营销页或游戏化界面。
- 应避免的气质：过度拟物、过度渐变、强烈霓虹、大面积高饱和色、可爱卡通、复杂装饰、重阴影。

## 色彩

优先使用 shadcn/ui 的 CSS variables 与 Tailwind token，不在业务组件中硬编码颜色。

- 主色：冷静的蓝紫或中性蓝
  - 用途：主按钮、选中态、当前单元格、焦点态、链接。
  - 建议 Hex：`#635BFF` 或使用主题变量 `--primary`。
- 辅助色：低饱和蓝灰
  - 用途：次级信息、hover surface、tag 背景。
  - 建议 Hex：`#EEF2FF` / `#E5E7EB`。
- 中性色：
  - 背景：`#FAFAFA`
  - 表面：`#FFFFFF`
  - 二级表面：`#F7F7F8`
  - 边框：`#E5E7EB`
  - 正文：`#18181B`
  - 辅助文字：`#71717A`
- 语义色：
  - 成功：用于已保存、完成率良好，不要大面积使用。
  - 警告：用于未保存、外部文件变更、覆盖风险。
  - 错误：用于保存失败、解析失败、API 错误。
  - 信息：用于导入完成、AI 建议生成完成。

## 深色模式

Phase 1（v0.1）可先以浅色模式为主，但样式应兼容 shadcn/ui 的 dark class。

- 深色背景：接近 `#09090B`
- 深色表面：接近 `#18181B`
- 深色边框：接近 `#27272A`
- 深色正文：接近 `#FAFAFA`
- 深色辅助文字：接近 `#A1A1AA`

## 字体

- 标题字体：系统无衬线，优先 `Inter`，回退到 `system-ui`、`-apple-system`、`BlinkMacSystemFont`。
- 正文字体：同标题字体。
- 等宽字体：`JetBrains Mono`、`SFMono-Regular`、`Menlo`、`Consolas`。
- 标题层级：
  - H1：20–24px，600 或 650 weight。
  - H2：16–18px，600 weight。
  - H3：14–15px，600 weight。
- 正文：13–14px。
- 表格正文：12–13px，允许高密度。
- 辅助文字：12px，颜色使用 muted foreground。

## 间距与布局

- 基础间距单位：4px。
- 页面布局：应用式全屏布局，而非文档流布局。
- 主布局：
  - 顶部 toolbar，高度约 44–52px。
  - 中央矩阵表格，占据主要区域。
  - 右侧或底部详情编辑区，用于单元格大文本编辑和 AI 建议。
  - 底部 status bar 可显示保存状态、筛选结果和统计摘要。
- 栅格：以表格列和 panel 为主要栅格，不使用复杂营销式 layout。
- 桌面端布局原则：优先信息密度和键鼠效率；横向滚动应稳定、可预期。
- 小屏原则：不优先适配移动端；Electron 桌面应用最低窗口宽度可设为约 1100px。

## 组件风格

### 按钮

- 主按钮：用于 Save、Apply AI Suggestion 等关键动作；高度 32–36px；圆角使用 shadcn 默认或 `rounded-md`。
- `Save PO/CSV` 与 `Save Project` 在对应内容存在未保存变化时显示轻量橙色提示点；两类保存状态彼此独立。
- 左下状态栏使用相同的独立保存状态：有未保存内容时列出 `PO/CSV not saved`、`Project not saved`，全部落盘后显示 `All changes saved`。最近一次源文件保存结果也固定显示在左下角：成功时说明已经写入并校验，失败时使用错误色并展示具体原因。
- 次按钮：用于 Open、Import、Filter、Stats 等工具动作。
- 危险按钮：仅用于覆盖、清空、移除文件等风险操作。
- 禁用态：降低 opacity，禁止 hover 强反馈。

### 输入框

- 默认态：边框清晰但低对比。
- 聚焦态：使用主色 ring，不能只靠颜色表达。
- 错误态：显示错误文案，不只显示红框。
- 搜索框：应始终容易访问，支持 source、任意语言和 tag 搜索。
- 顶部筛选菜单：使用轻量弹出菜单承载多选项；完成度筛选支持按语言多选，Source Tag 筛选支持多选，Word Tag 筛选同时支持 tag 多选和语言范围多选，均提供顶部 `All` 入口。
- Source Tag 和 Word Tag 筛选菜单中，`All` 与具体选项之间使用低对比虚线分隔；具体选项包含默认 `Empty`，用于定位没有标签的条目或单元格。勾选 `All` 时同步勾选全部具体 tag 与 `Empty`。

### 表格 / 矩阵

- 表格是主视觉组件。
- 表头 sticky。
- source 列可 sticky。
- source、语言和 tag 列宽可通过拖动表头分隔线调整；双击分隔线时按当前列内容自动适配。拖动时使用轻量主色分隔线反馈。
- 当矩阵总宽度超过可用区域时，仅矩阵区域横向滚动；右侧详情区保持在视口内。
- 语言列顺序可配置；表头使用用户设置的显示名称。
- 单元格文本默认单行截断，可通过详情区完整编辑。
- 当前选中单元格应有明确边框或背景。
- 语言表头可点击切换排序；激活列使用轻量强调和图标，仅允许一个语言列激活。
- 语言表头可直接拖拽重排；拖拽经过的目标表头使用轻量主色提示，不额外占用顶栏入口。
- changed 单元格使用轻量标记，例如左上角小点、淡色背景或细边框。
- 空译文使用 muted placeholder，不要使用强烈红色。
- 多选单元格使用低饱和选中背景和细边框；当前活动单元格仍使用更明确的主色边框，避免多选范围和编辑焦点混淆。
- tag 列支持直接输入和编辑 Source Tag；输入框保持低噪声、高密度样式，详情区同步提供较宽的 Source Tag 编辑入口。
- Word Tag 绑定具体语言单元格。单元格右上角预留少量 padding，使用细小圆角彩色竖条表示标签；多个标签并排显示多个竖条。相同标签颜色稳定一致，hover 时通过 tooltip 展示当前单元格的全部 Word Tag。

### 详情编辑区

- 详情区应像 Poedit 的大编辑框，但视觉上更现代。
- 必须清晰显示：source、当前语言、当前译文、Source Tag、Word Tag、AI suggestion。
- 当 key/context 真实存在时，在详情区显示；矩阵中可作为 source 下方的次级文字按需展示，避免长期占用空列。
- 多行文本框高度足够，避免用户只能在小单元格中编辑长文本。
- AI 建议必须是可应用的建议，不自动覆盖译文。

### 卡片与面板

- 卡片：轻边框、少阴影或无阴影。
- Panel：使用分隔线表达区域，不用厚重卡片堆叠。
- 弹窗：用于导入映射、保存确认、API 设置、批量替换预览。
- 设置类弹窗应使用较宽的内容宽度、安静的卡片分区和低噪声说明文字。AI 设置弹窗中，provider 与 DeepL Free / Pro 这类互斥选项使用分段卡片控件；连接信息与 prompt 分区展示，prompt 使用全宽输入区域，整体避免裸 radio 和密集表单堆叠。内容超过视口高度时，弹窗标题栏保持可见，内容区独立滚动。

### 导航

- Phase 1（v0.1）不需要复杂侧边栏。
- 顶部 toolbar + 可折叠右侧 panel 足够。
- 右上角使用紧凑的标准轻量弹出菜单切换应用界面语言；提供中文、日文和英文。弹层沿用顶部筛选菜单的边框、阴影、radio 选项和间距，不使用浏览器原生下拉样式。
- 统计可作为右侧 tab、底部弹层或轻量 panel。

## 图标与图像

- 应用图标（brand logo）：一个 L 形主体 + 右下 2×2 矩阵点，L 形从紫色渐变到青绿色，背景为深紫（`#1F1742` 一类）。
  - 含义：L 取自 Lingrid；矩阵点呼应“多语言 × 多 entry”的矩阵视图。
  - 应用图标统一使用 PNG；正式打包时由 electron-builder 自动生成 macOS `.icns` 和 Windows `.ico`。
  - 资源位置：
    - `assets/icon-source.png`：原始 1254x1254 源图，重新生成所有尺寸的 source of truth。
    - `assets/icon-{16,32,48,64,128,180,192,256,512,1024}.png`：常用尺寸。
    - `public/favicon.ico`：浏览器多尺寸 .ico（16/32/48/64/128/256）。
    - `public/favicon-16.png`、`public/favicon-32.png`、`public/apple-touch-icon.png`、`public/icon-192.png`、`public/icon-512.png`：浏览器 favicon 集合。
    - `public/site.webmanifest`：PWA / 安装元数据，背景色与图标主背景一致。
  - 重新生成或调整品牌时：修改 `assets/icon-source.png`，再由 `sips` 缩放各尺寸；`favicon.ico` 由 PIL 从 `icon-{16,32,48,64,128,256}.png` 拼合。
- UI 图标风格：使用 lucide-react 或 shadcn 常见搭配；线性、简洁、统一 stroke。
- 插图风格：Phase 1（v0.1）尽量不使用插图；空状态可以使用简单图标和短文案。
- 图片裁切：不适用。
- 空状态：强调下一步操作，例如“Open PO or Import CSV”。

## 动效

- 动效原则：只用于状态过渡、panel 展开、菜单出现和轻量反馈。
- 时长：120–180ms。
- 缓动：ease-out 或 shadcn/framer 默认轻量曲线。
- 禁止使用：大幅弹跳、复杂 loading 动画、炫技过场、影响表格操作效率的动效。

## 可访问性

- 对比度：正文与背景保持足够对比；辅助文字不能过浅。
- 焦点态：所有可交互元素必须有键盘焦点样式。
- 键盘导航：矩阵应逐步支持方向键移动、Enter 编辑、Esc 取消。
- 编辑提交：输入过程保持当前单元格稳定，失焦、Enter 或 Tab 后再提交；Enter 向下移动，Tab 横向移动，`Shift+Tab` 反向移动。
- 减少动态效果：尊重系统 reduced motion。
- 错误反馈：必须提供文字说明。

## Do / Don't

- Do：
  - 优先让用户一眼看见多语言并排关系。
  - 保持 toolbar 简洁。
  - 对保存状态、changed 状态、当前选中单元格给出明确反馈。
  - 在详情区提供舒适的大文本编辑体验。
  - 让 tag 像 Obsidian 一样轻量、可读、可搜索。
- Don't：
  - 不要把界面做成复杂本地化 SaaS 后台。
  - 不要在 Phase 1（v0.1）里塞入术语库、翻译记忆、协作、复杂 QA 的入口。
  - 不要让 AI 建议看起来像自动覆盖流水线。
  - 不要总显示空的 context 列。
  - 不要用花哨视觉压过表格本身。
